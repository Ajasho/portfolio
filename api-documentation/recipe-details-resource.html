Recipe Details (v1)

Use the Recipe Details resource to create, update, publish, and unpublish a recipe.

Create, update a recipe

https://api.abundo.osp.tech/externalapi/v1/{retailerBannerId}/recipe-details

Use this endpoint to create a recipe. If there is no recipe with equal retailerRecipeDetailsId then a new one is created, otherwise an existing one is updated. If a recipe refers to an unknown author, or recipe unit of measurement then the validation fails. The content type is recipesRecipeDetails.

path Parameters
retailerBannerId
required
string^[a-zA-Z0-9-_]{1,64}$

Your unique identifier for the retail banner.

header Parameters
x-api-key
required
string

The API key for your environment. If you don't know your API key, speak to your Ocado contact.

Authorization
required
string
Default: Bearer TOKEN

Format: Bearer {{token_value}}

RequestId
required
string
Default: aa13e836-5c1a-4b05-b24d-d30571c49da8

Request id

Request Body schema: application/json
retailerRecipeDetailsId
required
string^[a-zA-Z0-9-_]{1,64}$

Used to identify the recipe content when using the external API to manage your recipes. The value must be unique.

required
object

The title of the recipe details. The value is unique. The limit is 70 characters.

required
object

The description of the recipe details. No uniqueness restriction is applied. The value is limited to 256 characters.

required
object

The slug of the recipe details. The value is unique. The value is limited to 256 characters.

required
object

The image of the recipe details. The image size limit is set to 20MB. The image dimensions limits are 'width at most 10000px' and 'height at most 10000px'. The image asset is created every time an entry is created or update. A newly-created asset is pass to the Contentful Processing an assetAPI to trigger the image uploading into the Contentful storage.

required
object

The image alt text of the recipe details. No uniqueness restriction is applied. The value is limited to 256 characters.

publishedDate
required
string <date>

The published date of the recipe details.

retailerRecipeAuthorId
required
string^[a-zA-Z0-9-_]{1,64}$

The reatiler content id of the author linked to a given recipe details. The author should exist at the time of the recipes details entry processing.

required
Array of objects

The list of ingredients required to complete the recipe.

required
Array of objects

A set of steps the customer must follow to complete this recipe.

retailerRegionIds
Array of strings >= 0 items

The list of regions where the recipe details will be available in. The regions should be available under a given banner. If no retailer region ids are passed then all regions available under the given banner are assigned.

Array of objects

The list of terms used to search for the recipe, entered in lowercase. No uniqueness restriction is applied.

numberOfServings
integer <int32> >= 1

The number of servings in the recipe.

caloriesPerServing
integer <int32> >= 1

The number of calories in one serving.

preparationTimeMinutes
integer <int32> >= 1

The time required to prepare the ingredients.

cookingTimeMinutes
integer <int32> >= 1

The time required to cook the recipe.

totalTimeMinutes
required
integer <int32> >= 1

The total time required to prepare and cook the recipe.

Array of objects

Other terms that you want to link to the recipe, e.g. 'summer', 'Halloween', or 'quick', 'easy'.

seoIndex
boolean

Indicates whether the recipe collection details are indexed to appear in the Google search results:

  • true - index the recipe collection details
  • false - exclude from indexing The default value is null.
Array of objects

If multiple URLs are used to access this recipe, the canonical URL is the most representative one and will be crawled by Google.

Array of objects

The page title that is displayed in the browser's title bar or in the page's tab. If empty, the 'Recipe title tag rule' is used instead.

Array of objects

A short description of the page displayed in the search results. If empty, the 'Recipe description tag rule' is used instead.

Array of objects

The list of keywords to help describe the page to the legacy search engines.

Publish, unpublish a recipe

https://api.abundo.osp.tech/externalapi/v1/{retailerBannerId}/recipe-details/{retailerRecipeDetailsId}/published

Publishing a recipe regardless of its current state: draft, changed, published, or archived. Unpublishing a recipe only if its state is published. The content type is recipesRecipeDetails.

path Parameters
retailerBannerId
required
string^[a-zA-Z0-9-_]{1,64}$

Your unique identifier for the retail banner.

retailerRecipeDetailsId
required
string^[a-zA-Z0-9-_]{1,64}$

The unique identifier of the recipe content.

header Parameters
x-api-key
required
string

The API key for your environment. If you don't know your API key, speak to your Ocado contact.

Authorization
required
string
Default: Bearer TOKEN

Format: Bearer {{token_value}}

RequestId
required
string
Default: aa13e836-5c1a-4b05-b24d-d30571c49da8

Request id

Request Body schema: application/json
published
required
boolean

The field defines whether the entry content will be published or unpublished.

  • true - the unit of measurement is published
  • false - the unit of measurement is unpublished
