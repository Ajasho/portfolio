Product Propositions (v4)

Use the Product Propositions resource to upload and manage the customer-facing details of the products you sell.

The information referenced here is used in the online store and includes the description, images, product information details, the listing dates, and whether the product relates to any restriction groups.

Product propositions are retail banner and region-specific, but they reference the same basic product created with the Products resource. You can upload product propositions for a single region, or multiple regions.

Product proposition details are not made visible to customers until a new product catalog is generated.

You can also use product proposition overrides to change one or more attributes of a product. Overrides are the highest priority in the display hierarchy with the original Product proposition treated as a default layer.

For example, if the original product proposition is updated for a store, any values contained within the proposition override are still used in the online store.

It may take up to 2 hours for changes to age restriction to be realized in OSP. Because product data is fixed for an order just before finalization, this data must be updated before the finalization time to affect customer verification at the time of delivery or collection. If you require an urgent update to data, please raise a service desk ticket.

Create or update product proposition overrides for a product

https://api.abundo.osp.tech/externalapi/v4/{retailerBannerId}/products/{retailerProductId}/proposition-overrides

You can use this endpoint to override one or more product proposition attributes for a product.

When overriding the product proposition details, consider the following rules:

  • The attribute's new values replace the original ones, not merge with them. For example, if the tags array with ["refrigerated", "local"] is overridden with ["vegan"], the result of the override is ["vegan"].

  • To override values in an array, you must provide the entire set of values that constitute the array. For example, if the detailedInformation array includes five objects, and you want to add a new object, provide a total of six objects. To remove an object, provide four objects, removing the one you no longer need. To override an object, provide the entire set of values that constitute the object.

  • You don't need to provide the entire set of product proposition details. This means that if you want to override only listedDates, then in the request, provide the array with the listing dates, but exclude tags, deposits, or other details that are not changing. Any data that is not altered by an override will use the values from the original product proposition by default.

  • In the override request, provide either media or localizedMedia but not both.
