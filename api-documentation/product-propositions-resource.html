Product Propositions resource

Version 4
Base URL https:///externalapi/
Service account role WS Products
Sandbox mocks Yes
Status Active
Download JSON VIEW REFERENCE

Use the Product Propositions resource to upload and manage the customer-facing details of the products you sell.

The information referenced here is used in the online store and includes the description, assigned image IDs, product details, country of origin, tags, deposits, guaranteed product life, listing dates, and whether the product relates to any restriction groups.

Product propositions are retail banner and region-specific, but they reference the same basic product created with the Products resource. You can upload product propositions for a single region or for multiple regions.

Customers cannot see product proposition details until a new product catalog is generated.

You can also use product proposition overrides to change one or more attributes of a product. Overrides are the highest priority in the display hierarchy with the original Product proposition treated as a default layer.

For example, if the original product proposition is updated for a store, any value contained within the proposition override is still used in the online store.

It may take up to 2 hours for changes to age restrictions to be realized in OSP. Because product data is fixed for an order just before finalization, this data must be updated before the finalization time to affect customer verification at the time of delivery or collection. If you require an urgent update to data, please raise a service desk ticket.

Flexible units of measure

Units of measure are used in OSP to categorize and manage your product proposition. There are several units of measure, for example, “EACH”, “KG”, “L”, “CM”, “DOSE”. These are generally defined for a product by using the Product Storage resource, for example:

{
   …,    
   “pack”: {
      “itemAmount”: {
         “unit”: “EACH”,
         “value”: 42
      },
      “numberOfItems”: 1
   },
   …   
}

Using this information, OSP automatically calculates the pack size description, which in this case is: 42 per pack. The per pack element of this string derives from an internal, non-customizable mapping from pack.itemAmount.unit.

You can also specify the unit used for price comparisons, which acts as a unit price, for example, “EACH”, “PER_100G”, “PER_100ML”, “PER_1M”, and “PER_DOSE”. This is done using the Product Price resource, for example:

{
   …,
   “price“: {
      “amount”: 1.68,
      “currency”: “GBP”
   }
   “pricePer”: “EACH”,
   …
}

Using this information, OSP automatically calculates the unit price, which in this case is: £0.04 per item (1,68/42=0,04). The per item element of this string derives from an internal, non-customizable mapping from pricePer.

OSP additionally lets you configure flexible units of measure to effectively override these system defaults set for “EACH”. In other words, you can configure a custom variation of the per pack and per item expressions:

  • The per pack reflects the particular pack your item is sold in. For example, you can configure this to say ‘per box’ or ‘per carton’.
  • The per item reflects units that the pack consists of. For example, you can configure this to say: ‘per egg’, ‘per sachet’, or anything else that suits your business requirements.

You can use this feature to modify the per pack and per item elements of packed products only.

You can use any localized string to describe the package and unit price as long as it reflects the variation of ‘EACH’. In other words, it must not represent, for example, ‘WEIGHT’ or ‘VOLUME’ units.

Configuration

This feature is optional and the configuration requires using the Product Propositions resource.

The Product Propositions resource lets you configure the flexibleUnitsOfMeasure and decide on:

  • packUnitDescription to customise the per pack description string
  • unitPriceDescription to customise the per unit description string

You can configure both or only one of them. If you do not configure any custom description for these elements, OSP displays the system defaults described in the previous section. In case only one description is provided, it is displayed as configured, while the other one falls back to the system default.

Since the flexible units of measure allow you to customize the description string, you must ensure that both elements are variations of the system-defined “EACH” unit of measure.

If you use two inconsistent units of measure when setting up your product using, for example, “EACH” in the Product Storage resource and “PER_ 100ML” in the Product Price API, your product will not be displayed in the online store.

Mark products as ‘illegal to promote’

You can mark a product as ‘illegal to promote’ by using the illegalToPromote attribute under the restrictions attribute configuration in the Product Propositions resource.

After you set a product as ‘illegal to promote’, OSP excludes it from being available in promotions and other offers in your online store. This product is also filtered out and not displayed in the featured products campaigns.

Mark products as ‘restricted for review’

You can mark a product as ‘restricted for review’ by adding the reviewDisabled tag under the tags attribute in the Product Propositions resource.

After you mark a product as ‘restricted for review’, OSP no longer displays the Customer reviews section for that product so that customers cannot submit any reviews.

This helps you comply with the laws and regulations in your region that require you to restrict reviews on certain types of products, like medical products.

Guidelines for uploading HTML content

You can upload HTML content in the value attribute of detailedInformation to display the additional and extended details about a product, under different sections like Product information, Dietary information, and Nutritional data. Make sure that the HTML content you upload follows the Web Content Accessiblity Guidelines (WCAG) to make the content accessible to people with disabilities. For more information, refer to the following resources:

  • W3C web standards
  • Accessibility guidelines for HTML tables

Associate a product with a seller

The sellerId attribute lets you associate a product with a specific seller. You can configure another business entity to sell selected products.

This feature is optional. It requires extensive additional configuration and, depending on the scope you decide to adopt, potential further third-party integrations. Speak to your Ocado contact for more information.
